Mapping IDS alerts to the MITRE ATT&CK framework

Home

The objective of this project was to better understand how cyber attacks interact with Intrusion Detection Systems (IDS); specifically, to see exactly which packets trigger which IDS alerts, and then map these detections to the MITRE ATT&CK framework. To achieve this, I generated attack traffic using a vulnerabilty scanner; afterwards, I analysed the packets and IDS alerts in Wireshark, then mapped them to the MITRE ATT&CK framework.

MITRE ATT&CK Mapping
Tactic Reconnaissance (TA0043) & Initial Access (TA0001)
Technique Active Scanning (T1595)
Possible Tools/method Nessus, Burp Suite, Nmap Scripts, Nikto
Detection Source Snort alert -(SID 1564 -“WEB-MISC login.htm access”)
Impact Network reconnaissance or probing of web login pages (Enumeration). Possible precursor to a vulnerability scan or credential brute-force (severity: Medium)
Mitigations Network segmentation (e.g DMZ), Access control, Rate-limiting, MFA, etc
Snort IDS rule alert tcp $EXTERNAL_NET any -> $HTTP_SERVERS $HTTP_PORTS

Key skills applied & strengthened:

- Vulnerability scanning
- Alert validation
- IDS rules
- Using Snort Post-Dissector to integrate Snort IDS alerts directly into Wireshark packets
- Adversary mapping

This setup allowed me to instantly correlate intrusion alerts with the exact packets that triggered them and understand/modify IDS/IPS rules. I only included sufficient mappings necesssary for this demonstration, many more alerts were triggered during the scan and mapped in more detail.
